### Building Avuvo — intent-driven execution for the post-infra era

I'm Varun, a founder and engineer based in India. I build platforms where people run things without learning infrastructure — translating human intent into stable, running systems through intelligent, invisible operation.

Currently working on **[Avuvo](https://avuvo.com)**, an AI execution platform for the web.

---

**What I'm building**

- **[Avuvo](https://avuvo.com)** — intent-driven multi-agent DevOps. Plain language → shipped infrastructure. No YAML, no containers, no infra knowledge required.
- **[promptshield](https://github.com/varunmahajan1/promptshield)** — prompt injection defense for LLM agents. Zero dependencies. Pattern-based detection for user input, tool outputs, and file content.
- **[agent-stream](https://github.com/varunmahajan1/agent-stream)** — typed SSE event system for multi-agent orchestration. Agent handoffs, tool execution, thinking phases — all over Server-Sent Events.
- **[agent-runtime](https://github.com/varunmahajan1/agent-runtime)** — Go module with a Watchdog (stalled agent detection + 4-tier escalation) and Tool Enforcer (per-agent-type RBAC for tool calls).
- **[llm-meter](https://github.com/varunmahajan1/llm-meter)** — token cost tracking and sliding-window rate limiting for LLM apps. Redis or in-memory, zero config.
